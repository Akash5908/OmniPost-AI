# OmniPost AI - Implementation Plan

## Project Overview
Social media posting platform with AI transformation for LinkedIn and X (Twitter)

## Tech Stack
- **Frontend:** Next.js 15+ (App Router), TypeScript, NextAuth.js
- **Backend:** Node.js, Express, BullMQ + Redis
- **AI:** Vercel AI SDK / LangChain
- **Database:** MongoDB with Mongoose

---

## Phase 3: Backend API (Express + BullMQ)

### 3.1 API Endpoints

```typescript
// Routes
POST /api/posts          - Create post with platform selection
GET  /api/posts          - List user's posts
GET  /api/posts/:id      - Get single post
PUT  /api/posts/:id      - Update post
DELETE /api/posts/:id    - Delete post
POST /api/posts/:id/publish - Trigger queue job
```

### 3.2 Queue System (BullMQ)

```typescript
// Queue: social-post-queue
interface PostJob {
  postId: string;
  platform: 'linkedin' | 'x';
  content: string;
  accessToken: string;
}

// Processors
- linkedinPoster: Publish to LinkedIn API
- xPoster: Publish to X API
- retryHandler: Handle failed jobs with exponential backoff
```

### 3.3 Queue Configuration
- Concurrency: 5
- Retry attempts: 3
- Backoff strategy: exponential
- Remove on completion: false (keep for audit)

## Phase 4: AI Orchestration Layer

### 4.1 Strategy Pattern Implementation

```typescript
// ai/strategies/LinkedInStrategy.ts
- Expand on key professional takeaways
- Add professional formatting
- Include line breaks for readability
- Add relevant professional hashtags

// ai/strategies/XStrategy.ts
- Keep under 280 characters
- Punchy, engaging sentences
- 2-3 relevant hashtags
- No line breaks (compact format)

// ai/AIController.ts
- Route to appropriate strategy based on platform
- Use Vercel AI SDK for streaming responses
```

### 4.2 AI Service Interface

```typescript
interface AIStrategy {
  platform: 'linkedin' | 'x';
  transform(content: string): Promise<string>;
}

class AITransformService {
  transform(content: string, platforms: string[]): Promise<PlatformContent[]>;
}
```
